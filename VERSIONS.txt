KANAKU BOOK (SM TEX) — VERSION HISTORY
=======================================

The app version is shown at the bottom of the login screen and in the
top bar. It matches CACHE_NAME in sw.js; all three must be bumped
together whenever index.html changes (see README).


EARLY VERSIONS (v1 – v11)
-------------------------
Built up the core app, roughly in this order:
- Initial app: login, member buttons, amount + note + date entry,
  save to Google Sheet via Apps Script
- Stay logged in until explicit logout
- Developer credit on login page
- Ledger view: member totals, month/date search
- From–to date range search and tabular entries view
- Recent list shows last 10 entries from all users
- Gate saving on recent entries load; timezone, JSONP timeout and
  ledger drilldown fixes
- Opening splash animation with designer credit
- Service worker: serve latest page on every open, cache only as
  offline fallback
- Bigger designer credit on splash
- Entry delete from recent list, with confirmation
- Block new entries until a confirmed delete lands in the sheet
- Require typing "delete" to confirm entry deletion
